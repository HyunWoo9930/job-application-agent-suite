# Contributing

Thanks for contributing to this repository.

## Before You Open a PR

- Keep prompts and examples free of personal information
- Do not include real company-confidential data
- Prefer small, focused pull requests
- Update `README.md` and `README.en.md` when behavior or usage changes

## Good Contributions

- Better prompt design for a specific agent
- New safe sample inputs or sanitized demo outputs
- Improvements to corpus scanning heuristics
- Better QA checks for unsupported claims or repetitive tone
- Documentation improvements for first-time users

## Development Notes

- Scripts are plain Python and should stay lightweight
- Preserve UTF-8 support for Korean text
- Avoid adding heavy dependencies unless clearly needed
- Keep sample data sanitized and reusable

## Suggested PR Checklist

- I removed private or identifying information
- I tested the changed script or documented why I could not
- I updated related docs if needed
- I kept examples generic and safe to publish
